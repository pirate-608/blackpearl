# 选题一：AI Agent 框架搭建——从“会说话”到“能做事”
自 2023 年大语言模型（LLM）爆发以来，AI 的定位正从“聊天机器人”向“智能代理（Agent）”快速演进。虽然 LLM 具备强大的常识和推理能力，但它本质上是一个受限于训练数据的闭环系统——它无法感知实时天气、无法进行精确的数学运算，也无法操作你的本地文件。AI Agent 的出现打破了这层“次元壁”，通过引入 ReAct（推理+行动） 等范式，赋予了模型调用外部 API 和自主决策的能力，使其成为了人工智能通识教育中最前沿、最具工程价值的方向。AI Agent 框架搭建背景大语言模型（LLM）虽然能聊天和写文章，但本身不会上网、不会算数、不会操作文件——它只是一个"文字接龙"的模型。AI Agent 的核心思想是：让 LLM 学会"调用工具"和"自主决策"，从一个"会说话的模型"变成一个"能做事的智能体"。比如你问它"帮我查一下今天北京天气并写一首相关的诗"，Agent 会自己决定先调用天气 API、再把结果交给 LLM 写诗。这是 2023 年以来 AI 应用层最重要的范式之一，也是 ChatGPT、Claude 这些产品背后的关键技术。作业目标从零搭建一个简化版的 AI Agent 框架，让它能够：接收用户的自然语言指令 → 自主选择合适的工具 → 调用工具并获取结果 → 把结果整理后回复用户。整个过程支持多轮"思考-行动"循环。
## 核心技术要点：
1. Prompt 工程：设计一个能让 LLM 输出"工具调用指令"的系统提示词，参考 ReAct 范式（Reasoning + Acting，让模型交替输出"思考"和"动作"）
2. 解析与执行循环：解析 LLM 输出的 JSON 或特定格式，找到要调用的工具和参数，执行后把结果再喂回给 LLM，直到任务完成
3. 对话历史管理：维护一个messages列表，处理上下文长度限制

*备注：推荐技术栈Python + OpenAI / DeepSeek / 通义千问 等任意一家的 API（学生可以申请免费额度，或者用本地部署的开源小模型如 Qwen2.5-3B）。不允许直接用 LangChain、LlamaIndex 这类高层框架*

## 要求：
1.至少实现 3 个工具（如计算器、维基百科搜索、本地文件读写）
2.能完成一个需要多步调用的任务（比如"查一下爱因斯坦的出生年份，然后算一下他活了多少岁"）
3.提交代码 + 一个 demo 视频或对话截图

## 进阶（加分项）：
支持流式输出、加上短期记忆/长期记忆、做一个 Web 界面（Gradio/Streamlit）、实现多 Agent 协作（一个规划 Agent + 一个执行 Agent）。学到什么理解 LLM 应用层最核心的设计模式，建立"提示词即程序"的思维，熟悉 API 调用和 JSON 解析这些工程基本功。
